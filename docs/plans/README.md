# 计划目录说明

本目录用于保存多阶段计划。

## 目录约定

- `active/`：允许存在多个计划目录，用来保存当前未完成计划、最近完成计划和仍有参考价值的历史基线
- 同一时刻最多只能有一个“未完成执行计划”
- “未完成执行计划”必须在仓库根 `CLAUDE.md` 中明确指定；如果当前没有未完成计划，也必须显式写明
- 其他计划即使仍位于 `active/`，也只能视为默认基线或参考方案，不能被误判成待执行任务
- 后续如有需要，可增加 `done/`、`archived/` 等子目录

## 单个计划建议结构

- 一个计划一个文件夹
- 文件夹内至少包含：
  - `design.md`
  - `detailed-design.md`
  - `tasks.md`

## 任务状态约定

- `[ ]` 未开始
- `[x]` 已完成
- 如果需要人工补充说明，可直接在任务项后追加简短备注

## 执行约束

- 代理默认只能推进仓库根 `CLAUDE.md` 指定的未完成任务清单
- 如果仓库根 `CLAUDE.md` 明确写着“当前没有未完成执行计划”，则不要擅自把最近完成计划继续当成待办列表
- 完成实现后必须同步更新对应 `tasks.md`
- 如果发现任务不完整，应先补任务备注；如果这已经不是现有计划能承接的范围，就先新建或补充 plan，再继续实现
