# 计划目录说明

本目录用于保存多阶段计划。

## 目录约定

- `active/`：允许存在多个计划目录，但同一时刻只能有一个“唯一执行计划”
- “唯一执行计划”必须在仓库根 `CLAUDE.md` 中明确指定
- 其他计划即使仍位于 `active/`，也只能视为历史基线或参考方案，不能作为默认执行依据
- 后续如有需要，可增加 `done/`、`archived/` 等子目录

## 单个计划建议结构

- 一个计划一个文件夹
- 文件夹内至少包含：
  - `design.md`
  - `detailed-design.md`
  - `tasks.md`

## 任务状态约定

- `[ ]` 未开始
- `[x]` 已完成
- 如果需要人工补充说明，可直接在任务项后追加简短备注

## 执行约束

- 代理默认只能推进仓库根 `CLAUDE.md` 指定的唯一任务清单
- 完成实现后必须同步更新对应 `tasks.md`
- 如果发现任务不完整，应先补任务备注，再继续实现
